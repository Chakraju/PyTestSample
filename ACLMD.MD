# 4\. Topic Definitions

## 4.1 Source Topic - MyRetirement

The MyRetirement topic receives application events generated by the myRetirementApp platform. Each message represents a discrete customer interaction event captured by the retirement application.

| **Field Name**      | **Data Type**   | **Nullable** | **Description**                                                                      |
| ------------------- | --------------- | ------------ | ------------------------------------------------------------------------------------ |
| ApplicationSoftware | String          | No           | Identifier of the application software version or module that generated the event    |
| EventTS             | Timestamp (UTC) | No           | Timestamp of when the event occurred in the source system                            |
| CustomerID          | String          | No           | Unique identifier for the customer within myRetirementApp                            |
| ClientID            | String          | Yes          | Identifier for the client/employer associated with the customer's retirement account |
| EventTypeCd         | String (Code)   | No           | Coded value representing the category or classification of the event type            |
| EventResponseTxt    | String          | Yes          | Free-text response or outcome description associated with the event                  |

## 4.2 Destination Topic - Customer Interaction

The Customer Interaction topic is the canonical event stream consumed by CRM, analytics, and reporting platforms. It follows a standardized schema with nested event payload structures.

| **Field Name**          | **Data Type**        | **Nullable** | **Description**                                                                                     |
| ----------------------- | -------------------- | ------------ | --------------------------------------------------------------------------------------------------- |
| EventType               | String (Enum)        | No           | Standardized event type classification for the Customer Interaction domain                          |
| InteractionStartTS      | Timestamp (ISO-8601) | No           | Start timestamp of the customer interaction in ISO-8601 format                                      |
| CustomerID              | String               | No           | Unique customer identifier - carried through directly from source                                   |
| Event.ClientID          | String               | Yes          | Nested event object: client/employer identifier                                                     |
| Event.EventTypeCd       | String (Code)        | No           | Nested event object: original event type code from source system                                    |
| Event.EventResponseText | String               | Yes          | Nested event object: full text of the event response (field name expanded from source abbreviation) |

# 5\. Field Mapping Specification

The following table defines the authoritative field-level mapping between the MyRetirement source topic and the Customer Interaction destination topic. The ACL must implement all mappings as specified.

| **Source Field**    | **Destination Field**   | **Transform Type** | **Required** | **Transformation Rule**                                                                                                                                             |
| ------------------- | ----------------------- | ------------------ | ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ApplicationSoftware | EventType               | Derive / Map       | Yes          | Map ApplicationSoftware value to a standardized EventType enum. Maintain a configurable lookup table for this translation. Log unmapped values and apply a default. |
| EventTS             | InteractionStartTS      | Rename + Format    | Yes          | Rename field. Convert source timestamp to ISO-8601 format (yyyy-MM-dd'T'HH:mm:ss.SSSZ). Preserve UTC timezone.                                                      |
| CustomerID          | CustomerID              | Pass-through       | Yes          | Direct copy. No transformation required. Validate field is non-null and non-empty.                                                                                  |
| ClientID            | Event.ClientID          | Rename + Nest      | No           | Move into the nested Event object. Pass value as-is. If null in source, set null in destination.                                                                    |
| EventTypeCd         | Event.EventTypeCd       | Rename + Nest      | Yes          | Move into the nested Event object. Pass value as-is. Validate field is non-null.                                                                                    |
| EventResponseTxt    | Event.EventResponseText | Rename + Nest      | No           | Move into the nested Event object. Rename from abbreviated 'Txt' suffix to full 'Text'. Pass value as-is.                                                           |
